Frequently Asked Questions

1. What is the difference between var, let, and const?

The differences between var, let and const is given below:


  • var: Declares variables with function or global scope and allows re-declaration and updates within the same scope.

  • let: Declares variables with block scope, allowing updates but not re-declaration within the same block.

  • const: Declares block-scoped variables that cannot be reassigned after their initial assignment.
2. What is the difference between map(), forEach(), and filter()?

The difference between map(), forEach(), and filter() is given below:


  • map() : The map() function receives a function as a parameter and will apply the code on each element and returns an entirely new array. It will not change the original array.

  • forEach() : The forEach() function receives a function as an argument and it applies the same code to every element. It will not return anything, it just applies the conditions to every element. It will not change the original array.

  • filter() : The filter() method receives function as a parameter. It runs the function for each element in the array. It will return the new array which satisfies the condition applied. It will not change the original array.
3. Explain arrow functions and how they are different from regular functions?

Explaining arrow functions and how they are different from regular functions is given below :


Arrow Function: An arrow function is a shorter syntax for writing functions in JavaScript. Introduced in ES6, arrow functions allow for a more concise and readable code, especially in cases of small functions.


differences between arrow function and regular function : Arrow Function's syntax is shorter and more concise but the regular function needs the function keyword.
Arrow function can not be used as a constructor but regular function can be use as a constructor with new function.
Arrow function does't have its own arguments object but the regular function has its own object.

4. How JavaScript Promises work?

The Promise object represents the eventual completion (or failure) of an asynchronous operation and its resulting value.Now, how promises work in java script is given below :


  • Pending : This is the default state of a defined promise
  • Fulfilled : This is the state of a successful promise
  • Rejected : This is the state of a failed promise

A promise goes from pending to fulfilled or form pending to rejected . The fulfilled and rejected are indicate the end of a promise. In this way promises work in java script.

5. How closures work in JavaScript?

A closure is a function that allows access to variables from its outer function and global variables, even after the outer function has finished executing. This enables functions to “remember” their environment. In this manner closures work.
